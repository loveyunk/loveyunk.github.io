gzip压缩

第三方库CDN加速

  • 第三方的依赖文件可以使用cdn外链的方式引入，这样就能大大缩小项目文件的体积。

  1. 在 index.html 引入CDN

  2. 删除或注释依赖

  • 这些依赖以前是使用npm安装，现在需要在项目文件注释掉（或直接删除这些依赖），所有用到这些你需要替换的第三方依赖文件的代码都需要删除或注释
  1. 然后项目中需要改的地方是bulid文件夹下面的webpack.base.conf.js
  • externals 防止将某些 import 的包(package)打包到 bundle 中，而是在运行时(runtime)再去从外部获取这些扩展依赖(external dependencies)。
  • externals 作用是打包的时候不将依赖打包进去。如果一开始就是用script标签引入的，node_modules中本身就不存在，这里也可不用写。这里添加是为了早期使用npm安装，而后面要改为使用CDN的方式。
  • extrenals key 对应的是 from 'xxx', value对应的是 import 'xxx'
  1. 其他
  • 要注意：npm安装的小版本问题，要保证开发时 node_modules 里的版本库 与模板cdn中的第三方版本 要保持一致。
  • 可通过命令 npm run build --report 查看第三方依赖是否打包进去。
