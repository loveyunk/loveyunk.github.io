Vue ref用法总结

发表于 2018-06-03 | 分类于 Vue |

ref是reference的缩写表示引用。相当于给节点加一个ID引用，然后获取使用。

ref 被用来给元素或子组件注册引用信息。引用信息将会注册在父组件的 $refs 对象上。

Vue使得我们只需要操纵data，如何操纵DOM这不是我们关心的事，这也是vue与jquery最大的不同之处，而ref像一个逃生舱一样，让我们可以在万不得以的情况下直接去操纵底层DOM，但大部分情况下，你不应该这么做，而应该使用数据驱动的思想。

当然，我们也可以通过使用查询选择器来访问DOM元素来实现这样的效果，但是使用ref属性更简洁，而且这也是Vue中的方法。它也将更安全，因为你不会依赖于class和id。因此，几乎不会因为更改了HTML的标签或者CSS样式受到影响。

使用

  • 如果用在子组件上，则引用的是子组件实例（用在与子组件通讯，父组件获取子组件的数据）

  • 如果用在普通DOM元素上，引用的是DOM元素（等同于document.getElementById()）

More info: ref api

用处

  • 获取子组件数据
    • this.$refs.child.data 父组件获取子组件数据，但这不是响应式的。
    • 比如子组件是一个计时器，当用户在父组件执行某个操作后，父组件需要获取计时器时间向后台发送请求，此时可用$refs获取子组件时间。
  • 调用子组件方法
    当父组件执行某些操作后，执行子组件方法。

ref与v-for

用在元素节点上：获取的是DOM节点的数组。而通过document.getElementsByClassName获取的是HTMLCollection类数组，两者并不相同。但数组元素都是DOM节点，相对应的是相等的。

ref用在子组件上，获得的是组件实例的数组。

用在router-view上

对比vm.$children

  • vm.$children获取的是当前实例的直接子组件（子组件里面嵌套的子组件无法获取）。
  • $children并不保证顺序，也不是响应式的。
  • 获取的是一个子组件的数组。而$ref获取的是具体的某一个子组件的实例。

注意

  • $refs引用DOM元素或子组件在生命周期mounted可用，此时DOM挂载完成。
  • $refs不是响应式的。
  • 应该避免在模板或计算属性中访问 $refs。
  • 因为 ref 本身是作为渲染结果被创建的，在初始渲染的时候你不能访问它们 - 它们还不存在
